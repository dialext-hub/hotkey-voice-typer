# Дизайн: потоковая расшифровка по паузам (streaming chunks)

**Дата:** 2026-06-26
**Статус:** Утверждён, готов к плану реализации
**Файл:** `voice_typer.py`

## Проблема

Сейчас расшифровка линейная: пока удерживается горячая клавиша, аудио копится в
`recorder.frames`; после отпускания `process_audio()` сохраняет один WAV → одна
расшифровка → одна вставка. Пока клавиша зажата, на экране пусто — текст
появляется только после отпускания.

## Цель

Опциональный режим, в котором паузы в речи определяются прямо во время
удержания клавиши. При паузе накопленный кусок аудио уходит на расшифровку, и
текст печатается чанками по ходу речи, не дожидаясь отпускания.

## Решение

### Конфиг (новые поля, все опциональные)

```json
{
  "streaming": false,          // включает режим; по умолчанию старое поведение
  "pause_threshold": 0.8,      // секунд тишины = граница чанка
  "max_chunk_seconds": 15,     // принудительный флаш при долгой речи без пауз
  "silence_threshold": 0.01    // порог громкости (RMS, нормализованный 0..1), ниже = тишина
}
```

Хот-релоад как сейчас: значения читаются свежими (`load_config()`) в начале
каждой записи. `streaming` по умолчанию `false` — старое поведение не меняется.

### Компоненты

**`AudioRecorder` (расширяется) — детекция пауз (VAD).**
В режиме streaming `_callback` на каждом блоке аудио считает RMS-громкость и
сравнивает с `silence_threshold` (нормализованный RMS: `rms(int16) / 32768`).
Ведёт буфер текущего чанка + счётчик длительности тишины в хвосте + флаг
«была ли речь». Граница чанка срабатывает когда:

- накопилось ≥ `pause_threshold` секунд тишины **и** в чанке была речь → флаш, или
- длительность чанка ≥ `max_chunk_seconds` → флаш (если была речь).

Готовый чанк (numpy-массив) кладётся в очередь `ChunkTranscriber`. Чистая
тишина не флашится; слишком короткие всплески (< ~0.3 с накопленной речи)
не образуют чанк (защита от одиночных «бздыков»/кашля).

**`ChunkTranscriber` (новый) — один фоновый поток-воркер + FIFO-очередь.**
Ключ к правильному порядку текста: чанки расшифровываются и вставляются
**строго последовательно**, по одному. Если ответы по сети вернутся вразнобой —
порядок текста не нарушится. Цикл воркера:

1. достать чанк из очереди (блокирующе),
2. записать временный WAV,
3. `transcribe_groq()`,
4. `apply_replacements()`,
5. `paste_text()`.

Перед каждым чанком, кроме первого в одном утторансе (одно удержание клавиши),
вставляется пробел, чтобы слова на границе не слипались. Конфиг
(`api_key`, `proxy`, `paste_mode`, `voice_replacements`) читается свежим на
каждый чанк — как сейчас в `process_audio()`.

### Поток данных

```
KEY_DOWN → recorder.start(streaming=True) , tray=recording
   ... во время удержания: callback детектит паузы → кладёт чанки в очередь
       воркер параллельно расшифровывает и вставляет их по порядку
KEY_UP → recorder.stop() флашит финальный чанк , tray=processing
       когда очередь опустела → tray=idle
```

В нестриминговом режиме (`streaming: false`) поведение не меняется — прежний
путь `process_audio()` остаётся как есть.

### Краевые случаи и ограничения

- **Финальный чанк** при отпускании флашится всегда, если в нём была речь.
- **Автозамены через границу чанка** не срабатывают (фраза разбита на 2 чанка) —
  осознанное ограничение режима.
- **Качество** на коротких чанках чуть ниже, чем на цельной фразе — поэтому
  режим opt-in.
- **Буфер обмена** в clipboard-режиме перезаписывается на каждый чанк (как и
  сейчас на каждую запись).
- Порог `silence_threshold` зависит от микрофона — дефолт подобран средний, при
  необходимости крутится в конфиге.
- Состояние трея: `recording` пока зажата клавиша, `processing` после отпускания
  пока очередь не пуста, затем `idle`.

## Тестирование

- **Юнит-тест VAD-логики:** подаём синтетические массивы (тишина/речь/тишина) в
  логику детекции → проверяем границы чанков, счётчик тишины и флаг речи, без
  реального звука и сети.
- **Юнит-тест порядка:** мокаем `transcribe_groq` с разными задержками → проверяем,
  что вставка идёт строго по порядку очереди.
- **Ручная проверка:** «фраза — пауза — фраза» печатается двумя чанками; долгая
  речь без пауз — флашится по `max_chunk_seconds`; нестриминговый режим работает
  по-старому.
